# MCP 文档指南

## 构建命令
- 本地启动：`mintlify dev`
- 部署：合并 PR 到 main 分支后自动部署

## 风格指南
- 遵循现有的 MDX 格式和组件
- 保持内容清晰、简洁和技术准确性
- 在适当的地方包含实用的代码示例
- 提交前测试所有链接和代码示例
- 与现有文档保持一致的结构
- 记录 CLI 命令时，使用两行格式，提示符在单独的一行：`$ claude` 后跟 `> /command`
- 使用适当的标题：教程中使用"何时使用"、"步骤"和"提示"
- 分支命名约定：前缀为"ashwin/"（例如，"ashwin/add-mcp-cli-docs"）

## 文件结构
- 将新页面放在适当的部分（概念、教程等）
- 添加新页面时更新 docs.json
- 遵循现有的文件命名约定（kebab-case.mdx）
- 在 MDX 文件中包含正确的前置元数据

## 文档标准
- 优先考虑用户理解而非技术完整性
- 同时记录基本语法和具体示例
- 使用清晰的格式构建命令文档
- 对于图像分析示例，使用引号：`$ claude "分析这张图片：/path/to/image.png"`
